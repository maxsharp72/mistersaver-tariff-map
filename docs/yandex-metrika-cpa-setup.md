# Настройка Яндекс.Метрики для отслеживания CPA-конверсий

Это инструкция как отследить переходы по партнёрским ссылкам **MisterSaver Tariff Map** в Яндекс.Метрике, посчитать конверсии и понять с каких страниц приходят клики.

## Архитектура трекинга

```
Юзер на /tarify-zhku/chukotskij-ao/
            ↓ клик на «Твой кешбэк»
Браузер: GET https://mistersaver.ru/go/cashback/?ref=chukotskij-ao
            ↓
Сервер плагина:
    1) Логирует клик (admin → Tariff Map → Настройки → блок «Партнёрские клики»)
    2) Отдаёт 302 → https://go.leadgid.ru/aff_c?aff_id=139564&offer_id=7249&sub_id=chukotskij-ao
            ↓
LeadGid фиксирует переход с sub_id=chukotskij-ao
            ↓
Партнёр (банк) показывает форму заявки/регистрации карты
            ↓
Если юзер оформил карту → LeadGid начисляет CPA-вознаграждение
```

Мы можем отследить **клики** на сайте через Метрику и **конверсии** через кабинет LeadGid (sub_id скажет с какого региона пришёл клиент).

## Часть 1. Цели в Метрике для трекинга кликов по партнёркам

### Шаг 1.1 — Создать счётчик Метрики (если ещё нет)

1. Идём на [metrika.yandex.ru](https://metrika.yandex.ru)
2. **Добавить счётчик** → указываем `mistersaver.ru` (или `mistersaver-dev.ru` для тестирования)
3. Включаем **Вебвизор**, **Карты кликов**, **Карта скроллинга**
4. Копируем номер счётчика (например, `12345678`)

### Шаг 1.2 — Установить код Метрики

У тебя уже стоит плагин «Яндекс Метрика» — открой его настройки и убедись что номер счётчика тот самый.

Если код установлен вручную через тему — он должен быть в `<head>` всех страниц. Проверь через **F12 → Console → набери `ym`** — должна вернуться функция.

### Шаг 1.3 — Создать цели «Клик по партнёрке»

В Метрике → **Настройка → Цели → Добавить цель**.

Создаём **3 цели**:

#### Цель 1: «Клик по любой партнёрке» (общая)

- **Название**: `Партнёрский клик`
- **Тип условия**: JavaScript-событие
- **Идентификатор цели**: `partner_click`
- ✅ **Ретаргетинг**: включить (пригодится для рекламы догоняющим)

#### Цель 2: «Клик по кешбэк-карте»

- **Название**: `Кешбэк-карта (Твой кешбэк)`
- **Тип**: JavaScript-событие
- **Идентификатор**: `partner_click_cashback`
- ✅ Ретаргетинг

#### Цель 3: «Клик по Т-Банк (Оплатить ЖКУ)»

- **Название**: `Т-Банк Оплата ЖКУ`
- **Тип**: JavaScript-событие
- **Идентификатор**: `partner_click_tbank`
- ✅ Ретаргетинг

### Шаг 1.4 — Добавить JS-трекинг на сайт

В админке WP → **Внешний вид → Customizer → Дополнительные стили** (либо в **Astra → Custom Code** через Snippets-плагин) — добавь этот код:

```html
<script>
(function() {
  var COUNTER_ID = 17963905; // ← счётчик Метрики MisterSaver

  document.addEventListener('click', function(e) {
    var link = e.target.closest('a[data-offer]');
    if (!link) return;
    if (typeof ym !== 'function') return;

    var offer  = link.getAttribute('data-offer');    // cashback-card | tbank-zhku
    var region = link.getAttribute('data-region');   // moskva | chukotskij-ao | ''

    // Общая цель
    ym(COUNTER_ID, 'reachGoal', 'partner_click');

    // Конкретная цель по предложению
    if (offer === 'cashback-card') {
      ym(COUNTER_ID, 'reachGoal', 'partner_click_cashback');
    } else if (offer === 'tbank-zhku') {
      ym(COUNTER_ID, 'reachGoal', 'partner_click_tbank');
    }

    // Параметры визита — для среза по региону в отчёте «Параметры визитов»
    ym(COUNTER_ID, 'params', {
      partner_offer:  offer,
      partner_region: region || 'archive'
    });
  }, { capture: true });
})();
</script>
```

> **Счётчик `17963905` уже подставлен.** При переезде на другой сайт поменяй в первой строке (`var COUNTER_ID`).

### Шаг 1.5 — Проверка

1. Открой страницу любого региона
2. F12 → Console
3. Кликни на кнопку «Твой кешбэк» НАЖАТЫМ СРЕДНЕЙ КНОПКОЙ МЫШИ (или Ctrl+левый клик) — чтобы открылось в новой вкладке и текущая консоль не исчезла
4. В Наладках Метрики (нажать в трее счётчика «Flag» или открыть [Метрика Devtools](https://yandex.ru/support/metrica/quick-start.html#install-counter)) или просто в Console введи вручную:
   ```js
   ym(17963905, 'reachGoal', 'test_goal')
   ```
   — в Network-вкладке должен появиться запрос на `mc.yandex.ru/watch/...` (если AdBlock не блокирует)
5. Через 5-15 минут в Метрике → **Отчёты → Стандартные → Конверсии → Цели** появится цель «Партнёрский клик» с одним достижением

## Часть 2. UTM-метки для атрибуции трафика

Чтобы знать **с какого источника** пользователи кликают на партнёрки (поиск/реклама/прямой заход) — нужно проследить путь от источника до клика. Это **встроено** в Метрику автоматически через UTM-метки.

### Цепочка UTM

Когда пользователь приходит из:
- **Поиска Яндекса**: Метрика автоматически фиксирует Source/Medium = «Яндекс / организика»
- **Рекламы Яндекс.Директа**: автоматически по yclid-меткам
- **Реклама в РСЯ**: автоматически
- **Сторонние ссылки** (ВКонтакте, Тelegram, форумы) — нужно вручную ставить UTM на ссылках, которые публикуешь

Пример UTM при ссылке из ВК-поста:
```
https://mistersaver.ru/tarify-zhku/?utm_source=vk&utm_medium=post&utm_campaign=zhku_map
```

Когда такой посетитель кликнет «Твой кешбэк» — у тебя будет:
- В Метрике: Цель `partner_click_cashback`, источник `vk / post`, кампания `zhku_map`
- В отчёте LeadGid: `sub_id=chukotskij-ao` (если кликнул со страницы этого региона)

Сводя оба отчёта, ты увидишь: **источник трафика → страница региона → клик по партнёрке → конверсия в банке**.

## Часть 3. Воронка конверсии: от просмотра до клика

### Шаг 3.1 — Цель «Просмотр страницы региона»

В Метрике → **Цели → Добавить цель**:

- **Название**: `Просмотр страницы региона`
- **Тип**: URL-адрес страницы → `Содержит`
- **Условие**: `/tarify-zhku/` + дополнительный шаг (страница не равна архиву). Реальное условие:
  - Тип «**Составная цель**»
  - Шаг 1: URL содержит `/tarify-zhku/`
  - Шаг 2: URL **не равен** `/tarify-zhku/`

Это позволит отделить **просмотр конкретного региона** от просмотра главной карты.

### Шаг 3.2 — Цель «Просмотр главной карты»

- **Название**: `Просмотр главной карты`
- **Тип**: URL → точное совпадение
- **Условие**: `/tarify-zhku/` (или `/tarify-zhku`)

### Шаг 3.3 — Отчёт «Воронка конверсии»

В Метрике → **Отчёты → Конверсии → Достижение целей**:

1. Выбери цели в порядке воронки:
   1. `Просмотр главной карты` (или `Просмотр страницы региона`)
   2. `Партнёрский клик` (общая)
   3. `partner_click_cashback` ИЛИ `partner_click_tbank` (по отдельности)
2. Установи период — обычно «Последние 30 дней»
3. Сегментация — по регионам или источнику трафика

В итоге увидишь:
- **% пользователей**, которые после просмотра региона кликнули по партнёрке
- **Какие регионы** конвертируют лучше всего (Москва, Питер, Чукотка?)
- **С какого источника** идёт самый качественный трафик

## Часть 4. CPA-конверсии через LeadGid

Метрика покажет **клики** — это половина истории. **Реальная CPA-конверсия** (когда юзер оформил карту) фиксируется в кабинете партнёра.

### Шаг 4.1 — Кабинет LeadGid

1. Заходишь в [leadgid.ru](https://leadgid.ru) под своим аккаунтом (партнёр ID 139564)
2. **Статистика → По офферам**:
   - Оффер 7249 = кешбэк-карта (ПСБ)
   - Оффер 6432 = Т-Банк Black

### Шаг 4.2 — Срез по sub_id (=регион)

В фильтрах LeadGid установи **«Группировка по sub_id»**. Увидишь:

| sub_id | Клики | Лиды | Подтверждения | EPC | Выплата |
|---|---|---|---|---|---|
| chukotskij-ao | 142 | 12 | 8 | 67.2 | 5 600 ₽ |
| moskva | 980 | 45 | 32 | 38.5 | 38 400 ₽ |
| sankt-peterburg | 720 | 38 | 25 | 41.2 | 30 000 ₽ |
| ... | ... | ... | ... | ... | ... |

EPC (доход на клик) подскажет, в какие регионы вкладывать больше SEO-усилий.

### Шаг 4.3 — Сверка с Метрикой

Каждый ваш собственный лог (админка WP → MS Tariff Map → блок «Партнёрские клики») должен примерно соответствовать клиентам LeadGid. Если в Метрике 100 кликов, а в LeadGid 60 — значит 40 кликов либо:
- Прошли через AdBlock (но мы это уже обошли через `/go/`)
- Заблокированы провайдером
- Браузер не дошёл до партнёра по другим причинам

**Серверный лог плагина** обычно показывает самые точные цифры, потому что событие создаётся **до** редиректа, и AdBlock на него не влияет.

## Часть 5. Сводный дашборд

### Совет: настрой свой ритм проверки

**Ежедневно** (5 минут):
- WP → MS Tariff Map → Настройки → блок «Партнёрские клики». Сколько кликов вчера? Топ-3 региона.
- LeadGid → сводка по дню. Сколько лидов/подтверждений?

**Еженедельно** (15 минут):
- Метрика → Отчёты → «Конверсии» → «Источники, сводка». Какой источник дал больше всего `partner_click`?
- Метрика → «Карта кликов» на странице топ-региона. Куда пользователи кликают? Нет ли «слепых зон»?
- LeadGid → группировка по sub_id за неделю. Какие регионы дают лучший EPC?

**Ежемесячно** (1 час):
- Сравни в Метрике месяц/месяц: рост целей `partner_click_*`
- Если конверсии упали → проверь не появились ли новые блокировщики (uBlock обновляет фильтры регулярно), не уехала ли вёрстка
- Проверь не сменились ли URL партнёрок в LeadGid (если оффер закрыли — наши ссылки могут вести в 404)

## Часть 6. Что делать если конверсий нет

### Проверка 1 — Клики не фиксируются в плагине

WP → MS Tariff Map → Настройки → блок «Партнёрские клики». Если пусто — проблема в рендере или `/go/` не работает:

```bash
curl -I https://mistersaver.ru/go/cashback/
```

Должен быть `HTTP/1.1 302 Found` с заголовком `Location: https://go.leadgid.ru/...`. Если 404 — зайди в **Настройки → Постоянные ссылки** и нажми «Сохранить» (это пересоздаст rewrite-правила).

### Проверка 2 — Метрика не видит цели

В DevTools → Console → должна быть строка `ym(...)`. Если её нет:
- Проверь, что номер счётчика подставлен правильно в JS-сниппет
- Проверь, что Метрика подгружается (введи в console: `typeof ym`)
- Если AdBlock у юзера блокирует Метрику — серверный лог плагина всё равно покажет клик

### Проверка 3 — LeadGid не видит sub_id

Открой в браузере целевую ссылку с sub_id вручную:

```
https://go.leadgid.ru/aff_c?aff_id=139564&offer_id=7249&sub_id=test123
```

И посмотри в LeadGid после 10-15 минут — должен появиться клик с `sub_id=test123`. Если нет — напиши в саппорт LeadGid.

## Технические детали (для разработчиков)

**Где в коде ссылки**: `templates/parts/partner-cta.php` (рендер) и `includes/class-redirector.php` (обработчик).

**Данные на ссылках**:
- `data-offer="cashback-card"` или `data-offer="tbank-zhku"` — идентификатор оффера
- `data-region="moskva"` (или пусто на архиве) — slug региона

**Серверный лог**:
- Хранится в `wp_options.ms_tariff_partner_clicks` (последние 500 записей)
- Агрегированная статистика в `wp_options.ms_tariff_partner_stats`
- Очистить можно из админки или через PHP: `MS_Tariff_Map_Redirector::reset_log()`
